# CodeGuard Server Module Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the CodeGuard FastAPI server that stores review results from the Plugin and serves them to the Web UI via a versioned REST API, backed by PostgreSQL.

**Architecture:** Repository pattern with an ABC (`ReviewRepository`) and a concrete `PostgresReviewRepository` using SQLAlchemy async + asyncpg. FastAPI handles routing with Pydantic schemas for validation. Alembic manages migrations. API Key auth via a dependency injection guard. Docker Compose orchestrates postgres + server containers.

**Tech Stack:** Python 3.12, FastAPI, SQLAlchemy 2.0 (async), asyncpg, Pydantic 2, pydantic-settings, Alembic, pytest, pytest-asyncio, httpx, uvicorn, Docker, PostgreSQL 16.

```python
from datetime import datetime
from typing import Any
from uuid import UUID
from pydantic import BaseModel, Field
from app.models.finding import FindingCreate, FindingResponse

class ReviewCreate(BaseModel):
    version: str = Field(..., min_length=1)
    summary: dict[str, Any]
    files_changed: list[str]
    findings: list[FindingCreate] = Field(default_factory=list)

class ReviewResponse(BaseModel):
    id: UUID
    project_id: UUID
    version: str
    summary: dict[str, Any]
    files_changed: list[str]
    created_at: datetime
    model_config = {"from_attributes": True}

class ReviewDetailResponse(ReviewResponse):
    findings: list[FindingResponse] = Field(default_factory=list)
```
